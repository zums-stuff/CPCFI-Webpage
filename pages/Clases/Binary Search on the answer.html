Búsqueda Binaria sobre la Respuesta

¿Qué es Binary Search on the Answer?

Es una técnica de optimización que usa búsqueda binaria sobre un rango de posibles respuestas a un problema, en lugar de sobre una estructura de datos como un arreglo.

  • Se aplica a problemas donde necesitamos encontrar un valor máximo o mínimo que cumpla una cierta condición.
  • La idea clave es transformar el problema de "encontrar un valor" a un problema de decisión: "¿Es un valor "k" una respuesta válida?".
  • Si podemos responder esa pregunta de manera eficiente, podemos encontrar la solución óptima en tiempo logarítmico.

La Clave: Una Función Monótona 📈

Para que esta técnica funcione, el problema debe tener una propiedad monótona. Esto significa que si una respuesta "k" es válida, todas las respuestas menores que "k" (o mayores, dependiendo del problema) también deben ser válidas.

Por ejemplo, en el problema de encontrar el "n" máximo tal que n² < X, la función de decisión es f(n) = (n² < X).

  • Si f(8) es "True" (porque 64 < X), entonces f(7), f(6), &dots; también serán "True".
  • Si f(11) es "False" (porque 121 ≥ X), entonces f(12), f(13), &dots; también serán "False".

Esta propiedad nos permite descartar la mitad del espacio de búsqueda en cada paso.

Ejemplo: Encontrar n máximo tal que n² < X

Dado un entero positivo "X", queremos encontrar el máximo entero positivo "n" que satisfaga n² < X. Por ejemplo, para "X = 101", la respuesta es 10, ya que 10² = 100 < 101, pero 11² = 121 no es menor que 101.

En lugar de probar "n = 1, 2, 3, ..." (lo que sería lento, O(√X)), podemos buscar binariamente la respuesta en el rango "[0, X]", logrando una complejidad de O(log X).

Implementación en C++:


#include <iostream>

// Función de decisión (predicado monótono)
// Devuelve true si n² < X
bool check(long long n, int X) {
    return n * n < X;
}

int main() {
    int X = 101;
    long long low = 0, high = X;
    long long ans = 0;

    while (low <= high) {
        long long mid = low + (high - low) / 2;

        if (check(mid, X)) {
            // 'mid' es una respuesta válida.
            // La guardamos y probamos con un valor más grande.
            ans = mid;
            low = mid + 1;
        } else {
            // 'mid' no es una respuesta válida.
            // Probamos con un valor más pequeño.
            high = mid - 1;
        }
    }

    std::cout << "The maximum n is: " << ans << std::endl;
    return 0;
}

Pasos para Aplicar la Técnica 📝

  1. Definir el Espacio de Búsqueda [L, R]: Identifica los límites inferior y superior para la posible respuesta.
  2. Crear la Función de Decisión "check(k)": Esta función debe ser monótona y determinar si un valor "k" es una solución válida.
  3. Implementar el Bucle de Búsqueda Binaria: Usa la función "check(k)" para ajustar los límites "L" y "R" hasta encontrar la respuesta óptima.

Problema de Ejemplo: Codeforces 1221C - Perfect Teams

Tienes "c" estudiantes programadores, "m" matemáticos y "x" sin especialización. Un equipo "perfecto" tiene 3 miembros, con al menos 1 programador y 1 matemático. ¿Cuál es el máximo número de equipos perfectos que puedes formar?

La función "check(k)" sería: "¿Es posible formar "k" equipos?". Si se pueden formar "k" equipos, se pueden formar "k-1". Si no se pueden, tampoco se podrán "k+1". Esta es la propiedad monótona que nos permite usar Binary Search on the Answer.

Implementación en C++:


#include <bits/stdc++.h>
using namespace std;

// check(k): ¿Podemos formar k equipos?
bool canMake(long long k, long long c, long long m, long long x) {
    // Requerimos k programadores y k matemáticos.
    // El resto de estudiantes (c-k, m-k, y x) deben sumar
    // lo suficiente para completar los k equipos de 3.
    if (c < k || m < k) return false;
    return (c - k) + (m - k) + x >= k;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int q;
    cin >> q;
    while (q--) {
        long long c, m, x;
        cin >> c >> m >> x;

        // El número de equipos no puede ser mayor que el min(c, m)
        long long low = 0, high = min(c, m), ans = 0;

        while (low <= high) {
            long long mid = low + (high - low) / 2;
            if (canMake(mid, c, m, x)) {
                ans = mid;
                low = mid + 1; // Intentar formar más equipos
            } else {
                high = mid - 1; // No se puede, intentar con menos
            }
        }
        cout << ans << "\n";
    }
    return 0;
}
