Algoritmo de Floyd-Warshall

¿Qué es el Algoritmo de Floyd-Warshall?

Es un algoritmo de programación dinámica que encuentra el camino más corto entre todos los pares de nodos en una gráfica (All-Pairs Shortest Path).

  • Funciona en gráficas dirigidas y no dirigidas.
  • Maneja aristas con pesos positivos y negativos.
  • Su única limitación es que la gráfica no puede tener ciclos negativos (un ciclo cuya suma de pesos sea menor que 0).

La Idea Central: El Vértice Intermedio

El algoritmo funciona iterativamente, tratando a cada vértice k (de 0 a V-1) como un posible nodo intermedio en el camino de i a j.

La pregunta clave que se hace es: "Para ir de i a j, ¿es más corto ir directo, o es más corto pasar por k?"

Esto se traduce en la siguiente línea de código, que es el corazón del algoritmo:

// ¿Es el camino actual de i a j
// más largo que el camino de i a k + el camino de k a j?
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]);

Definición Formal de la DP

Aunque la implementación no lo parezca, Floyd-Warshall es un DP puro. El estado real de la DP se define así:

dp[k][i][j] = "La distancia más corta desde el nodo i hasta el nodo j, usando solamente vértices intermedios del conjunto {0, 1, ..., k}."

La transición es:

dp[k][i][j] = min( dp[k-1][i][j], dp[k-1][i][k] + dp[k-1][k][j] )
  • dp[k-1][i][j]: El camino más corto sin usar k.
  • dp[k-1][i][k] + dp[k-1][k][j]: El camino más corto usando k.

Como dp[k] solo depende de dp[k-1], podemos optimizar el espacio y eliminar la primera dimensión (k), quedando solo la matriz dist[i][j] que se actualiza en el lugar. El orden de los bucles es crucial: el bucle de k debe ser el más externo.

Inicialización de la Matriz

Para que el algoritmo funcione, la matriz de distancias dist[n][n] debe inicializarse correctamente:

  • dist[i][j] = INF (un valor muy grande) si no hay una arista directa de i a j.
  • dist[i][i] = 0 para toda la diagonal (la distancia de un nodo a sí mismo es 0).
  • dist[i][j] = w si hay una arista directa de i a j con ese peso.

Implementación en C++

Nota: Esta implementación es para una gráfica dirigida. 


#include<bits/stdc++.h>
using namespace std;

const long long INF = 1e18;

int main(){
    int n, m, q;
    cin >> n >> m >> q; // n=vértices, m=aristas, q=consultas

    // Inicializa la matriz con INF
    vector<vector<long long>> dist(n, vector<long long>(n, INF));
    
    // La distancia a uno mismo es 0
    for (int i = 0; i < n; i++) {
        dist[i][i] = 0;
    }
    
    // Lee las aristas
    for(int i = 0; i < m; i++){
        int a, b;
        long long c;
        cin >> a >> b >> c;
        a--; b--; // Convertir a 0-indexado
        
        // Asigna el peso de la arista dirigida
        // Se usa min por si hay aristas paralelas (múltiples aristas entre a y b)
        dist[a][b] = min(dist[a][b], c);
    }

    // El corazón del algoritmo de Floyd-Warshall
    for(int k = 0; k < n; k++){ // Vértice intermedio
        for(int i = 0; i < n; i++){ // Nodo de inicio
            for(int j = 0; j < n; j++){ // Nodo final
                
                // Chequeo para evitar overflow con INF
                if(dist[i][k] < INF && dist[k][j] < INF) {
                    dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]);
                }
            }
        }
    }

    // Responde las consultas
    while(q--){
        int a, b; cin >> a >> b;
        a--; b--;
        if(dist[a][b] == INF) // Importante checar con == INF
            cout << -1 << "\n";
        else 
            cout << dist[a][b] << "\n";
    }
}

Complejidad y Casos de Uso

La complejidad temporal es siempre O(V³), debido a los tres bucles anidados.

Esta complejidad no depende del número de aristas (M). Por esto, Floyd-Warshall es ideal para gráficas densas (donde M es cercano a V²).

En gráficas ralas (donde M es cercano a V), es más eficiente ejecutar V veces el algoritmo de Dijkstra (O(V * (E log V))).

Aplicaciones Adicionales

Detección de Ciclos Negativos

Floyd-Warshall puede detectar si existe un ciclo negativo en la gráfica.

Después de que el algoritmo ha terminado, simplemente revisa la diagonal de la matriz. Si para cualquier vértice i, el valor de dist[i][i] es menor que 0, significa que existe un camino desde i de regreso a i con costo negativo, es decir, un ciclo negativo.

Reconstrucción de Caminos

Para reconstruir el camino más corto (no solo la distancia), se usa una matriz auxiliar next[i][j].

  • Inicialización: Si hay una arista i -> j, next[i][j] = j.
  • Actualización: Cuando relajas la arista (i, j) usando k, actualizas el camino: 
    
if (dist[i][k] + dist[k][j] < dist[i][j]) {
    dist[i][j] = dist[i][k] + dist[k][j];
    next[i][j] = next[i][k]; // El camino de i a j ahora va por k
}

Algoritmo de Bellman-Ford

¿Qué es el Algoritmo de Bellman-Ford?

Es un algoritmo que calcula el camino más corto desde un único nodo fuente a todos los demás nodos en la gráfica (Single-Source Shortest Path).

  • Su principal ventaja sobre Dijkstra es que funciona correctamente con aristas de peso negativo.
  • Su principal aplicación es detectar ciclos negativos.
  • Es más lento que Dijkstra, con una complejidad de O(V·E) (donde V es el número de vértices y E el de aristas).

La Idea Central: Relajación Iterativa

La idea de Bellman-Ford es simple: "relaja" todas las aristas de la gráfica una y otra vez.

Relajación: Es el mismo proceso que en Floyd-Warshall. Para una arista (u, v) con peso w, comprueba:

// ¿Es el camino actual a 'v' más largo que
// el camino a 'u' + el costo de ir de 'u' a 'v'?
if (dist[u] + w < dist[v]) {
    dist[v] = dist[u] + w;
}

El algoritmo repite este proceso para **todas las aristas** un total de V-1 veces.

  • Un camino más corto en una gráfica sin ciclos puede tener como máximo V-1 aristas.
  • En la primera iteración, encuentra todos los caminos más cortos de longitud 1.
  • En la segunda iteración, encuentra todos los caminos más cortos de longitud 2 (usando los resultados de la primera).
  • ...
  • Después de V-1 iteraciones, ha encontrado todos los caminos más cortos posibles.

Implementación en C++

Bellman-Ford generalmente se implementa sobre una lista de aristas, ya que es lo más fácil para iterar sobre todas ellas.


#include<bits/stdc++.h>
using namespace std;

const long long INF = 1e18;

// Es más limpio usar una struct para la lista de aristas
struct Edge {
    int a, b;
    long long cost;
};

int main(){
    int n, m;
    cin >> n >> m; // n=vértices, m=aristas

    vector<Edge> edges;
    for(int i = 0; i < m; i++){
        int a, b;
        long long c;
        cin >> a >> b >> c;
        a--; b--; // 0-indexado
        edges.push_back({a, b, c});
    }

    int source_node = 0; // Nodo fuente
    vector<long long> dist(n, INF);
    dist[source_node] = 0;

    // Bucle principal de Bellman-Ford: V-1 iteraciones
    for(int i = 0; i < n - 1; ++i) {
        // Iterar sobre TODAS las aristas
        for (const auto& e : edges) {
            if (dist[e.a] != INF) { // Si el nodo 'a' es alcanzable
                dist[e.b] = min(dist[e.b], dist[e.a] + e.cost);
            }
        }
    }
    
    // (Aquí iría la detección de ciclo negativo)
    
    // Imprimir distancias desde el nodo 0
    for(int i = 0; i < n; ++i) {
        if(dist[i] == INF) {
            cout << "INF ";
        } else {
            cout << dist[i] << " ";
        }
    }
    cout << endl;
}

Detección de Ciclos Negativos

La característica más poderosa de Bellman-Ford es cómo detecta ciclos negativos.

La lógica es simple: si después de las V-1 iteraciones (que garantizan el camino más corto si no hay ciclos negativos), realizamos una iteración más (la V-ésima) y alguna distancia *todavía* se puede mejorar, es porque esa arista es parte de un ciclo negativo.

Código de Detección

Se añade este bucle después del bucle principal de V-1 iteraciones:


bool negCycle = false;
for (const auto& e : edges) {
    if (dist[e.a] != INF && dist[e.a] + e.cost < dist[e.b]) {
        negCycle = true;
        break;
    }
}

if (has_negative_cycle){ //Hay ciclo negativo
    //...
}
